cd - смена директории 
dir -Force -Recurse - список файлов текущего каталога 
c отображением скрытых и системных элементов (-Force)
с отображением вложенных файлов и каталогов (-Recurse)


git clone ссылка - скачать репеозиторий к себе 
git remote - список настроенных удалённых репозиториев 
git remote -v - с адресами 
origin - репозиторий (сервер) с моего аккаунта 
upstream - источник форка 

git init . - создать гит репозиторий в текущей директории 
git init \ - с путём 
git status - информация о текущей ветке и изменениях
git branch - информация о ветках и выделение текущей 
git branch new_branch - новая ветка 
git branch -d - удалить ветку, надо уйти с этой ветки 
git chekout - перемещение по веткам 
git checkout -b new_branch - новая ветка и перемещение в неё
git checkout номер коммита (узнать из git log) - переход на другой коммит, возврат на старые версии
git commit --amend (нажимаем insert и меняем сообщение коммита, затем Esc и пишем :wq!)
git reset --hard HEAD~2 - удалить 2 последних коммита 
git reset --soft HEAD~3 - удалить из лога 3 последних коммита, при этом файлы останутся последней версии

 


git add * - все файлы добавились в stage статус для коммита 
git commit -m "My Initial commit, this is Version1.0" - сделан коммит(сохранение/снимок) с сообщением
изменения сохранены в локальном репозитории 
git push origin - сохранить в мой удалённый репозиторий
git merge - объединение веток, указать им ветки, добавляемой к главной, находится в главаной  
git push origin --delete имя ветки - удаление удалённого репозитория 

Статусы файлов: 
Untracked (создан файл) -> Staged (git add выбран для коммита) -> Copy in Local Repository (git commit)
-> Copy in Remote Repository (git push)


git log - выдаёт все записанные коммиты
git log -1 - последний коммит 
git log -1 -p - подробности о изменениях в последнем коммите 
git checkout -- назавание файла - восстановление файла после изменения 
git diff --staged - разница между последним коммитом и текущими изменениями в файле


.gitignore - просто файлы, папки и расширения, которые будут игнорироваться гитом
